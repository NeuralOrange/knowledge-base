# About Me

## Who I Am

I'm a software engineer passionate about software architecture, system design,
and building things that last.

## What This Site Is

This is my **Knowledge Base** — not a blog, but a public digital garden.

It works like this:

```
Obsidian (private) → curation → MkDocs (public)
```

I write notes in Obsidian. When a note matures — when I've thought about it,
applied it, and refined it — it graduates to this site.

Some pages are **evergreen** :material-leaf: — they're never "done" and grow
with my experience. Others are **completed** :material-check: — stable reference
material.

## Why This Exists

1. **Learn in public** — writing clarifies thinking
2. **Build a knowledge graph** — connections matter more than individual posts
3. **Create a portfolio** — not just code, but thinking
4. **Help others** — what I learn might help someone else
